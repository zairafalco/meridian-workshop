## 1. Introduction

Meridian Components is a mid-market distributor of industrial automation parts (sensors, actuators, controllers, circuit boards, power supplies) operating warehouses in San Francisco, London, and Tokyo. Annual revenue is approximately $9.6M.

We are seeking a qualified vendor to **modernize and extend our existing inventory management dashboard**. The current system was delivered by a previous vendor in 2024 and is functional but incomplete. Source code and the previous vendor's handoff notes are included with this RFP package so respondents can scope accurately.

## 2. Background & Current State

Our operations team relies on a web-based dashboard to track stock levels, orders, supplier spend, and demand forecasts across warehouses. The current system:

- Runs as a Vue/Python web application hosted internally
- Covers core inventory, orders, and spending views
- Has **known defects in the Reports module** that the previous vendor did not resolve before contract end
- Lacks several capabilities our operations team has requested (see §3)
- Has **no automated test coverage**, which has made our IT team reluctant to approve changes

The previous vendor's documentation is limited. We expect the selected vendor to perform their own architecture review as part of onboarding.

## 3. Scope of Work

The selected vendor will deliver the following. Items are listed in Meridian's priority order.

### 3.1 Required

| # | Requirement | Description |
|---|---|---|
| R1 | **Reports module remediation** | Audit and resolve all defects in the existing Reports page. Our team has logged at least eight issues including filter behavior, internationalization gaps, and inconsistent data patterns. |
| R2 | **Restocking recommendations** | New capability: a Restocking view that recommends purchase orders based on current stock levels, demand forecast, and an operator-supplied budget ceiling. |
| R3 | **Automated browser testing** | Establish end-to-end test coverage for critical user flows so our IT team can safely approve future changes. |
| R4 | **Architecture documentation** | Deliver a current-state architecture overview suitable for handoff to Meridian IT. Format at vendor's discretion. |

### 3.2 Desired (evaluated but not mandatory)

| # | Requirement | Description |
|---|---|---|
| D1 | **UI modernization** | Refresh the visual design to align with current standards. |
| D2 | **Internationalization** | Extend i18n support to remaining modules (Tokyo warehouse staff currently work in English-only views). |
| D3 | **Dark mode** | Operator-selectable theme. Several warehouse floor stations run in low-light environments. |

## 4. Proposal Requirements

Responses should include:

1. **Executive summary** — your understanding of our needs and proposed approach (1 page max)
2. **Technical approach** — how you will address each item in §3, including any assumptions
3. **Relevant experience** — comparable engagements
4. **Timeline** — phased delivery plan
5. **Pricing** — fixed-fee or T&M with not-to-exceed

A short capabilities presentation (10–15 slides) may be requested from shortlisted vendors.
