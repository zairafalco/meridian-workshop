R4 Architecture Documentation

Current-state architecture overview for Meridian Components' inventory management dashboard.

System Overview

This application is a two-tier, single-page dashboard built with a Vue 3 frontend and a Python FastAPI backend. The frontend is responsible for UI rendering, filter management, and API integration, while the backend exposes REST endpoints that read mock JSON data from the repository.

Frontend Architecture

  • Framework: Vue 3 with Composition API.
  • Build tool: Vite via client/package.json and npm run dev.
  • Entry point: client/src/main.js bootstraps the app and mounts App.vue.
  • Views: client/src/views/ contains the main pages, including Reports.vue, Dashboard.vue, and inventory/order views.
  • Components: reusable UI pieces live in client/src/components/, such as filter controls, modals, and navigation elements.

The frontend uses axios to call backend APIs at http://localhost:8001/api and a shared composable in client/src/composables/useFilters.js to keep filter state consistent across views.

Backend Architecture

  • Framework: FastAPI with Pydantic models.
  • Runtime: Uvicorn, started by python -m uvicorn main:app --reload --host 0.0.0.0 --port 8001.
  • Application file: server/main.py defines REST endpoints and common filter helpers.
  • Data models: Pydantic classes such as InventoryItem, Order, and DemandForecast validate API responses.
  • CORS: configured to allow cross-origin access from the frontend development host.

The backend currently implements endpoints for inventory, orders, demand, backlog, summary dashboards, spending, and reports. The report endpoints now support shared filter parameters to match the UI filter controls.

Data Flow & Storage

  • Data source: JSON files in server/data/ provide the current dataset.
  • Read-only: the system is currently read-only for production workflows; no database or persistence layer is included in scope.
  • API aggregation: backend endpoints aggregate and transform the JSON data for dashboard and report views.
  • Front-end filtering: filters are passed from the shared UI state to backend endpoints via query parameters.

Deployment & Startup

  • Backend startup: cd server and run python -m uvicorn main:app --reload --host 0.0.0.0 --port 8001.
  • Frontend startup: cd client and run npm install, then npm run dev.
  • Default ports: frontend on 3000 (or another free port if 3000 is occupied), backend on 8001.
  • Dependencies: Node packages in client/package.json and Python dependencies in server/requirements.txt.
Note: The current architecture is optimized for quick local development and review. For production deployment, a containerized or cloud-hosted model with a database backend would be the next recommended evolution.

Handoff Summary

This document describes the current-state architecture with the existing Vue 3 frontend, FastAPI backend, and JSON-based data layer. It is suitable for Meridian IT to understand the application shape, developer workflow, and where to extend the system.

Key extension points for future work:

  1. Replace JSON data with a relational or NoSQL database for persistence.
  2. Add authentication/authorization to API routes.
  3. Move frontend and backend into a CI/CD pipeline with automated tests per R3.
